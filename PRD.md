## Code-First, Visually Minded

**Core belief:** Code is just 1s and 0s if it doesn't have visual uniqueness to it.

When I first learned to code, it felt cold and sterile. No life. No soul. I had to learn to customize my environment - themes, fonts, layouts - before I could build with joy. Once the visual experience matched my creative standards, building became better.

**This is ACiD's approach:**

- The chain works (code-first)
- But everything you see and touch should inspire (visually minded)
- Explorer UI that doesn't look like every other Blockscout fork
- Bridge experience that feels intentional
- Documentation that's a pleasure to read
- Developer tools that spark creativity

**The thesis:** Web3 has incredible builders who lack visual inspiration. If they were more visually inspired, they'd be happier building. Happy builders build better. Better builders make better products for everyone.

ACiD isn't just an L2. It's a statement: **infrastructure can have soul.**

**Anyone can build on ACiD.**

The founder isn't the chain. The founder just launched the idea. ACiD is permissionless ‚Äî for builders, creators, degens, agents, anyone. No gatekeepers. No permission needed. Just build.

**We are not promising:**
- Revolutionary technology (it's a fork)
- Token price appreciation (there might not even be a token)
- Changing the world
- Being better than Base

**We are promising:**
- A working L2
- The joke will land
- It'll be fun to build

## Tech Stack (Keep It Simple)

| Component | What We Use | Why |
|-----------|-------------|-----|
| Rollup Framework | OP Stack (Bedrock) | Battle-tested, same as Base |
| Execution | op-geth | EVM compatible, no changes needed |
| Sequencer | Single operator (us) | Decentralize later if it matters |
| Bridge | Standard OP bridge | Don't reinvent the wheel |
| Explorer | Blockscout | Open source, works fine |
| RPC | Self-hosted initially | Alchemy/QuickNode later if needed |

**Modifications from vanilla OP Stack:**
- Chain ID
- Branding/naming
- That's basically it
